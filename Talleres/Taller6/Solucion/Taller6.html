Si observamos los rango de los valores podemos ver que tienen diferentes escalas y rangos. Podriamos mencionar que H2OC (mmol/mol) y VPact (mbar) muestran un rango estrecho con alta concentración de valores en su distribución, lo que podría ser indicativo de una variable más homogénea y con menos variabilidad.

Para preprocesar los datos y extraer las secuencias con las que se entrenara la red puede usar la siguiente funcion:

In [7]:
def generator ( data , lookback , delay , min_index , max_index , step =6) :
    if max_index is None :  
        max_index = len( data ) - delay - 1
    i = min_index + lookback
    rows = np . arange (i , max_index )
    samples = np . zeros (( len ( rows ) , lookback // step ,data . shape [ -1]) )
    targets = np . zeros (( len ( rows ) ,))
    for j , row in enumerate ( rows ):
        indices = range ( rows [j] - lookback , rows [j ], step )
        samples [j] = data [ indices ]
        targets [j] = data [ rows [ j] + delay ][1]
    return samples , targets

La funcion genera los inputs (variable “samples”) junto con los respectivos valores de la variable respuesta, la temperatura (variable “targets”).

Los parametros de la funcion estan descritos de la siguiente forma (Chollet, Francois. Deep learning with Python. Simon and Schuster, 2021 ):

• data — The original array of floating-point data.

• lookback — How many timesteps back the input data should go.

• delay — How many timesteps in the future the target should be.

• min index and max index — Indices in the data array that delimit which timesteps to draw from. This is useful for keeping a segment of the data for validation and another for testing.

• step — The period, in timesteps, at which you sample data. You’ll set it to 6 in order to draw one data point every hour.

Arquitectura de la Red Neuronal:

  • Capa de Entrada (Input): Esta capa define la forma de la entrada. El parámetro shape=(None, float_data.shape[-1]) indica que la red recibirá datos con un número variable de pasos de tiempo (por eso None), pero con un número fijo de características en cada paso de tiempo

  • Primera Capa Recurrente (SimpleRNN): Esta es una capa recurrente que tendra una salida de esta capa tendrá 32 valores por cada paso de tiempo.

return_sequences=True: Esto indica que esta capa devuelve las secuencias completas de salidas, es decir, las salidas para todos los pasos de tiempo, lo cual es necesario porque la siguiente capa SimpleRNN también es una capa recurrente. Y su función de activación ReLU (Rectified Linear Unit) es utilizada aquí.

  • Segunda Capa Recurrente (SimpleRNN): Es una capa recurrente que tiene los mismo parametros que la primera capa

  • Capa Densa de Salida (Dense): Esta capa densa es la capa de salida. Tiene una sola neurona (1) porque estamos realizando una predicción de un solo valor continuo (por ejemplo, temperatura). No se especifica una función de activación,

Compilación del Modelo (model.compile): El modelo usa el error absoluto medio (MAE) como función de pérdida. Esta red usa el optimizador Adam con una tasa de aprendizaje de 0.001. Adam es un optimizador avanzado que ajusta la tasa de aprendizaje para cada parámetro, y durante el entrenamiento, el modelo también mide el error cuadrático medio (MSE)

Ejercicio 5¶

  1. Investigue sobre las unidades GRU (Gated recurrent unit).

Las unidades GRU (Gated Recurrent Units) son un tipo de capa recurrente en redes neuronales, diseñadas para abordar el problema de desvanecimiento del gradiente en secuencias largas, similar a las LSTM (Long Short-Term Memory), pero con una estructura más simple.

Características clave de las GRU:

  • Las GRU utilizan dos puertas principales; La puerta de actualización, esta controla qué cantidad de información del estado anterior se debe mantener y; La puerta de reinicio, esta decide cuánta de la información pasada debe "olvidarse" al generar el nuevo estado.
  • Dentro de sus ventajas es en un mejor rendimiento en tareas de procesamiento secuencial, como el análisis de series temporales o procesamiento de lenguaje natural. Y tambien implica en un menor consumo de memoria y computación que las LSTM.
  • Usualmente estas unidades GRU se usan frecuentemente en tareas como traducción automática, análisis de sentimientos, predicción de series temporales, y otros problemas que involucran secuencias largas de datos. Son especialmente útiles cuando se necesita un modelo eficiente y rápido para procesar datos secuenciales.
  1. y 3. Para implementar las unidades GRU o LSTM puede usar: “layers.GRU” o “layers.LSTM” en lugar de “layers.SimpleRNN”. Y experimente con dichas unidades y describa sus resultados. Comparelos con aquellos obtenidos con las capas layers.SimpleRNN.

Los modelos más complejos, como GRU, LSTM, y Bidireccionales, podrían no estar bien optimizados para los datos o necesitar más épocas de entrenamiento, ya que presentan un rendimiento inferior, pero debido a ciertas limitaciones computacionales y tiempos de procesamiento se dejo de esta manera. Entonces la ReLU destaca aquí, es importante considerar el tipo de problema y los datos utilizados, ya que en otros contextos los modelos recurrentes más avanzados como GRU o LSTM suelen superar a las activaciones estándar como ReLU.
